# Processing Flow

## Canonical stage flow

```mermaid
flowchart TD
    A[User submits one Bilibili VOD URL] --> B[Create task, stages, and one durable TaskJob]
    B --> C[ingest]
    C --> D[media_prep]
    D --> E[asr]
    E --> F[translation]
    F --> G[highlight]
    G --> H[export stage marked skipped\nawaiting user-confirmed clip export]
    H --> I[report]
    I --> J[Task detail page shows artifacts and workspace]
    J --> K{User confirms a highlight candidate?}
    K -->|yes| L[POST /api/tasks/{task_id}/clips]
    L --> M[ffmpeg exports MP4 clip to /data/tasks/<task_id>/exports]
    K -->|no| N[Task keeps subtitles, report, and candidate data only]
```

Notes:

- The worker is single-job and durable.
- The pipeline runs the canonical stages in order.
- In the current MVP, clip export is a manual follow-up action after the report exists.
- A zero-candidate result is still a successful `highlight` stage.

## Artifact tree

```text
/data/
├── tasks.sqlite3
└── tasks/
    └── <task_id>/
        ├── raw/
        │   └── source.mp4
        ├── work/
        │   ├── asr-input.wav
        │   ├── asr-alignment-raw.json
        │   ├── asr-segments.json
        │   ├── subtitles.zh.srt
        │   ├── subtitles.zh-ja.json
        │   ├── subtitles.zh-ja.srt
        │   └── highlight-candidates.json
        ├── exports/
        │   └── clip-<start_ms>-<end_ms>.mp4
        ├── reports/
        │   └── task-report.md
        └── logs/
            ├── ingest.log
            ├── media_prep.log
            ├── asr.log
            ├── translation.log
            ├── highlight.log
            ├── export.log
            └── report.log
```

Notes:

- `exports/` stays empty until the user confirms a clip.
- `reports/task-report.md` is generated by the pipeline even if no clip is exported.
- `highlight-candidates.json` can contain ranked candidates or a `no_candidates` explanation.
- The artifact metadata in SQLite points back to these disk paths.
