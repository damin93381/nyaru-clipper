# 用户手册

## 工作站概览

Nyaru-Clipper 会在 `/` 直接打开桌面端**任务库**。这是面向单一操作员的工作站，用于处理已结束的 Bilibili VOD，以及主机已经可见的媒体文件。任务库、队列、任务概览、复核工作区和恢复操作都位于同一套三栏驾驶舱中。

当前 MVP 同一时间只运行一个受 GPU 约束的流水线任务。不支持直播录制、自动发布切片、账号体系或公网部署。

## 新建任务

在命令栏选择**新建任务**。抽屉会先校验来源，随后才能创建任务。

- **Bilibili 录播**：粘贴 HTTPS 的 `bilibili.com` 链接（可使用获准子域），且路径必须为 `/video/BV…`；选择**检查来源**，确认标题和上传者预览，再选择 Standard 与优先级。工作站会明确拒绝短链接和 HTTP 链接，不会解析它们的跳转目标。
- **本地文件**：只能浏览操作员配置的导入根目录。选择受支持媒体文件后，可选**引用原始文件**或**复制到任务存储**。浏览器只会拿到不透明的根目录 ID 和相对目录路径，绝不会拿到主机绝对路径。

创建后，任务会进入有序的单 GPU 队列，并打开 `/workstation/tasks/<task_id>`。

## 操作任务库和队列

任务库支持搜索、状态/来源/日期/产物就绪状态筛选、标签、排序和分页，以便处理大量任务。选择行即可在固定的右侧上下文面板中检查单个任务，或执行受支持的批量操作。

**处理队列**是手动排序界面。只有等待中的任务可以移动：可拖动队列行，也可使用支持键盘操作的菜单。如果其他操作先改动了队列，工作站会恢复后端权威顺序，并明确提示本次排序没有生效。

## 跟踪任务和恢复失败

任务概览展示七个标准阶段：

1. `ingest`
2. `media_prep`
3. `asr`
4. `translation`
5. `highlight`
6. `export`
7. `report`

选择某个阶段会更新安全日志和产物检查器。当后端提供恢复操作时，概览只显示该操作。例如，缺少 ASR 模型时可以下载模型，然后从 ASR 重新尝试；某个失败阶段也只能从服务器批准的阶段重试。上游产物会继续保留。

连接横幅会显示实时工作站事件状态。事件流连续失败后，界面会保留当前快照，并每 15 秒刷新一次；事件流恢复后会回到实时更新。

## 复核和导出

任务概览中保留原有复核工作区：

- 在受控滚动的表格中检查中文字幕和日语字幕；
- 复核排序后的高光候选，并调整开始/结束秒数；
- 选择**确认导出**，为确认的区间创建一个 MP4；
- 产物就绪后下载字幕、报告和已导出的片段。

零候选仍代表高光阶段成功：即使没有可导出片段，字幕和报告也可能仍可下载。

## 受信任局域网边界

只能在受信任的局域网或主机本机使用工作站。该产品刻意没有认证、TLS 终止、限流、租户隔离或公网暴露防护。不要做端口转发，也不要直接暴露到互联网。

## 路由兼容性

`/` 是工作站任务库。`/workstation`、`/workstation/queue` 和 `/workstation/tasks/<task_id>` 都可直接访问。旧的 `/tasks/<task_id>` 链接会重定向到对应的工作站任务概览。
